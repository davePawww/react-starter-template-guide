Deploy to Vercel

Why Vercel?

Vercel offers the best developer experience for deploying Vite apps. It automatically detects your framework, optimizes builds, provides global CDN with automatic SSL, and offers instant rollbacks. Made by the creators of Vite!

Method A: CLI Deployment

bash 
# Install Vercel CLI
pnpm add -g vercel

# Deploy
vercel

Follow the prompts - Vercel will automatically detect Vite and configure build settings.

Method B: Git Integration (Recommended)

  1. Push your code to GitHub
  2. Go to vercel.com and sign up
  3. Click Add New... ‚Üí Project
  4. Import your GitHub repository
  5. Vercel automatically detects Vite - just click Deploy
‚úÖ Vercel Settings (Auto-detected)
  • Framework Preset: Vite
  • Build Command: pnpm run build
  • Output Directory: dist

Automatic Deployment with Vercel + GitHub Actions

Deploy automatically to Vercel every time you push to main.

Step 1: Get Vercel Credentials

  1. VERCEL_TOKEN: Create at vercel.com/account/tokens
  2. VERCEL_ORG_ID: Run pnpm exec vercel whoami --json
  3. VERCEL_PROJECT_ID: Get from Vercel project settings

Step 2: Add Secrets to GitHub

  1. Go to your repo ‚Üí Settings ‚Üí Secrets and variables ‚Üí Actions
  2. Click New repository secret
  3. Add VERCEL_TOKEN
  4. Add VERCEL_ORG_ID
  5. Add VERCEL_PROJECT_ID

Step 3: Create Deploy Workflow

.github/workflows/deploy.yml 
name: Deploy to Vercel

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Run linter
        run: pnpm run lint

      - name: Run tests
        run: pnpm run test:run

      - name: Build project
        run: pnpm run build

      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
